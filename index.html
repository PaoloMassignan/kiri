Kiri is an on-premises proxy that intercepts every outgoing LLM call and strips proprietary source code before it reaches the API. Zero config changes to your existing tools.

How it works

Three-level filter pipeline

Every outgoing LLM call passes through three independent checks. Each level catches a different class of leak — from exact symbol matches to semantic similarity to extraction intent.

L2 — always active

Symbol matching

Word-boundary regex against your protected symbol list. Catches any prompt that names a protected class or function verbatim. Never fails silently.

REDACT
L1 — semantic

Vector similarity

Your source is indexed with all-MiniLM-L6-v2 and stored in a local ChromaDB. Cosine similarity ≥ 0.90 triggers redaction even when symbols are renamed.

REDACT
L3 — intent classifier

Ollama classifier

Prompts scoring 0.75–0.90 enter a grace zone and are classified by a local qwen2.5:3b model. Blocked only when explicit extraction intent is detected.

BLOCK

REDACT replaces protected function bodies with a stub comment before forwarding — the developer still gets a useful response. BLOCK (HTTP 403) is reserved for explicit extraction attempts detected by L3. The embeddings index stores float vectors only — source text never leaves your machine.

Known limits

What Kiri does not catch

We document the gaps honestly. Understanding the limits is as important as knowing what works.

  • 📝
    Semantic reformulation (pure prose) A developer describing what the code does in plain English — without naming symbols or pasting code — often scores below the L1 threshold. The embedding model lacks enough signal. This is the primary documented gap.
  • 📏
    Very short or highly generic snippets Method bodies shorter than ~4 lines, or code using only common patterns (getters, trivial constructors), may not produce a distinctive enough vector to cross the similarity threshold.
  • 🔌
    GitHub Copilot Copilot uses a hardcoded endpoint (copilot-proxy.githubusercontent.com) that cannot be redirected to a custom gateway. There is no workaround at the client side.
  • 🔑
    Claude Pro / Max subscriptions These plans use OAuth session tokens instead of static API keys. Kiri cannot intercept traffic authenticated this way. A standard Anthropic API account is required. On the roadmap.
  • ⚠️
    Fail-open on L1/L3 errors By design: if the embedding service or Ollama classifier is unavailable, requests are passed through rather than blocked. L2 symbol matching always remains active as a safety net.
