How it works

Three-level filter pipeline

Every outgoing LLM call passes through three independent checks, each catching a different class of leak — from exact symbol names to semantic code similarity to explicit extraction intent.

L2 — always active

Symbol matching

Word-boundary regex against your protected symbol list. Catches any prompt that names a protected class or function verbatim. Never fails silently — active even when L1 and L3 are unavailable.

REDACT
L1 — semantic

Vector similarity

Your source is indexed with all-MiniLM-L6-v2 and stored in a local ChromaDB. Cosine similarity ≥ 0.90 triggers redaction even when class names are renamed or code is partially pasted.

REDACT
L3 — intent

Ollama classifier

Prompts scoring 0.75–0.90 enter a grace zone and are classified by a local qwen2.5:3b model. Blocked only when explicit extraction intent is detected. Runs entirely on your hardware.

BLOCK if intent

Embeddings store float vectors only — source text never leaves your machine. On errors, Kiri fails open (PASS) rather than blocking work — L2 remains the safety net.

Known limits

What Kiri does not catch

We document the gaps honestly. The benchmark results above reflect these limits in practice.

📝
Semantic reformulation — pure prose

A developer describing code behaviour in plain English, without naming symbols or pasting code, often scores below the L1 threshold. The embedding lacks enough signal. This is the primary documented gap (see benchmark FN cases above).

📏
Very short or generic snippets

Method bodies shorter than ~4 lines, or code using only common patterns (trivial getters, constructors with no distinctive logic), may not produce a vector distinctive enough to cross the threshold.

🔌
GitHub Copilot

Copilot uses a hardcoded endpoint that cannot be redirected to a custom gateway. There is no client-side workaround.

🔑
Claude Pro / Max subscriptions

These plans use OAuth session tokens instead of static API keys. A standard Anthropic API account is required. On the roadmap.

⚠️
Fail-open by design

If the embedding service or Ollama classifier is unavailable, requests are passed through rather than blocked. L2 symbol matching always remains active as the safety net. This is an explicit design decision — documented in ADR-004.
