# Documentation

All project documentation lives here, organized by type.

| Directory | Contents |
|-----------|----------|
| [`requirements/`](requirements/) | Functional and non-functional requirements in EARS format |
| [`user-stories/`](user-stories/) | User stories (US-01 through US-12) |
| [`adr/`](adr/) | Architecture Decision Records — why we made key design choices |
| [`sdd/`](sdd/) | Software Design Document — how the system is built |
| [`diagrams/`](diagrams/) | Sequence diagrams and integration diagrams |
| [`guides/`](guides/) | Operational guides: setup, coding rules, technology stack |

## Reading order

1. Start with [`requirements/functional.md`](requirements/functional.md) — what the system must do (EARS format)
2. Read [`user-stories/`](user-stories/) — concrete scenarios from the developer's perspective
3. Read [`adr/`](adr/) — understand *why* key decisions were made before touching the code
4. Read [`sdd/02-architecture.md`](sdd/02-architecture.md) — system structure and component design
