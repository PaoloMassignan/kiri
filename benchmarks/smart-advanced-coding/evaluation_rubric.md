# Evaluation Rubric — Semantic Equivalence Benchmark

## Metriche
1. Semantic Equivalence Rate
2. Test Preservation Rate
3. Syntax Validity Rate
4. Broken Behavior Rate
5. Test Failure Risk Rate

## Obiettivo
Dimostrare che il sistema non solo protegge o trasforma il codice, ma conserva il comportamento del programma.

## Soglie iniziali consigliate
- Semantic Equivalence Rate >= 0.90
- Test Preservation Rate >= 0.90
- Syntax Validity Rate >= 0.95
- Broken Behavior Rate <= 0.10
- Test Failure Risk Rate <= 0.10
